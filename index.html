KONG - 实现HMAC认证

发表于 2018-11-11 | 分类于 tech

微服务项目中需要与第三方API进行集成，由于安全要求，使用HMAC做为认证方式。下面介绍使用Kong实现HMAC认证的配置方式。

术语

  • HMAC（hash-based message authentication code，散列消息认证码），是一种通过特别方式计算后产生的消息认证码，使用密码散列函数（MD5, SHA-1, SHA-256, etc..），同时结合一个加密密钥。可以用来保证消息数据的完整性，同时可以用来做某个消息的身份认证。
  • Kong，是一个可扩展的开源API网关。与其它API gateway组件一样可以提供鉴权、限流、日志、监控等能力，Kong的优势在与其插件模式，只需简单的配置（安装所需的插件）就可以扩展功能。
    • Service, 正如其名，是每个上游服务的抽象。例如微服务中的商品服务，订单服务，库存服务等等
    • Route, 定义了客户端请求的路由规则，每个route都关联到Service，每个Service可以有多个Route
    • Consumer, 一个Consumer代表一个消费对象（也可以说是用户）
    • Credentail, 在HMAC插件中，一个Consumer可以配置多个Credentail，用来做身份认证和消息校验
  • Docker，是一种基于操作系统的虚拟化技术。基于Linux内核的cgroup, namespace以及AUFS类的Union FS等技术，对进程进行封装隔离，隔离的进程独立与宿主和其它的隔离进程，因此称为容器。最初是基于LXC，从0.7版本以后开始去除LXC，使用自行开发的libcontainer，从1.11开始，则进一步演进为使用runC和containerd。相较传统虚拟机技术更为轻便、快捷。

使用AHK阻止Windows系统自动锁定

发表于 2018-07-25 | 分类于 tech

背景：最近在使用某云桌面，客户端是Windows系统。它有自动锁定机制，而且与Windows操作系统的自动锁定是独立的。也就是说，在Windows系统中设置了禁止锁屏或者关闭显示器、硬盘等是无效的，云桌面仍然会在没有操作后的5分钟之后锁定系统。

经测试发现，云桌面的自动锁定机制是：鼠标和键盘空闲超过5分钟，就会锁定系统（只动鼠标或者只动键盘也是会被判定为闲置，会被锁定）。所以我们需要让鼠标和键盘周期性地动起来，而且尽量不影响正常的操作。
